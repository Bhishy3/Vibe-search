# Contributing to Vibe Search

Thank you for your interest in contributing to Vibe Search! 🎉

## How to Contribute

### Reporting Bugs

1. Check existing [Issues](https://github.com/Bhishy3/Vibe-search/issues) to avoid duplicates
2. Open a new issue using the **Bug Report** template
3. Include your OS version, Firefox version, and steps to reproduce

### Suggesting Features

1. Open a new issue using the **Feature Request** template
2. Describe the feature and how it fits the project's AI-first mission
3. Include mockups or references if possible

### Pull Requests

1. **Fork** the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes and test with `npm start`
4. Commit with clear messages: `git commit -m "feat: add page translation"`
5. Push and open a Pull Request against `main`

## Development Setup

```bash
git clone https://github.com/YOUR_USERNAME/Vibe-search.git
cd Vibe-search
npm start
```

## Code Style

- Use consistent indentation (2 spaces)
- Comment complex logic in extension scripts
- Keep CSS selectors specific to avoid conflicts with Firefox internals
- Test all `prefs.js` changes against [Cover Your Tracks](https://coveryourtracks.eff.org/)

## Code of Conduct

Be respectful, inclusive, and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/) Code of Conduct.
